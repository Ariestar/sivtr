# 0.2.6 - 2026-07-13

## English

### Added

- Added `sivtr setup` for first-time install: pick shells, install hooks, migrate workspace keys, install MCP into selected agent hosts, then run a smoke check. ([2eda454](https://github.com/Ariestar/sivtr/commit/2eda454), [4b68fec](https://github.com/Ariestar/sivtr/commit/4b68fec))
- Added shared interactive prompts (`dialoguer`) used by `setup`, `doctor --fix`, and `share`. Shell install is multi-select; MCP host install is multi-select. ([4b68fec](https://github.com/Ariestar/sivtr/commit/4b68fec))
- Added `sivtr doctor --fix` and `sivtr doctor --json`. `--fix` can create config, install shell hooks, migrate workspace keys, and install MCP for detected hosts; interactive mode confirms before applying. ([59aa756](https://github.com/Ariestar/sivtr/commit/59aa756), [4b68fec](https://github.com/Ariestar/sivtr/commit/4b68fec))
- Expanded MCP install targets beyond Claude/Cursor/Codex to OpenCode, Pi, Hermes, and Cursor through a unified `AgentProvider` model (`sivtr mcp install --target ...`). ([606907a](https://github.com/Ariestar/sivtr/commit/606907a))
- Added dry-run workspace key inspection (`inspect_workspace_keys`) so doctor reports migrations without applying them until `--fix`. ([4b68fec](https://github.com/Ariestar/sivtr/commit/4b68fec))
- Added contributing guide, WeChat sponsor page, and richer install/troubleshooting docs. ([0a5362e](https://github.com/Ariestar/sivtr/commit/0a5362e))

### Changed

- Absorbed `sivtr migrate` into `sivtr doctor --fix` / `sivtr setup`; the standalone migrate command is gone. ([59aa756](https://github.com/Ariestar/sivtr/commit/59aa756))
- `sivtr init` no longer prompts to start the Windows Codex hotkey; start it yourself with `sivtr hotkey start` when needed. ([4b68fec](https://github.com/Ariestar/sivtr/commit/4b68fec))
- Default cargo-binstall / install path prefers musl Linux assets where applicable. ([0a5362e](https://github.com/Ariestar/sivtr/commit/0a5362e))
- Dropped Homebrew packaging from the release matrix and docs. Prefer `cargo binstall`, `install.sh`, or `install.ps1`. ([9bf9651](https://github.com/Ariestar/sivtr/commit/9bf9651))
- Removed remaining legacy `sift` compatibility paths and docs. ([48cef98](https://github.com/Ariestar/sivtr/commit/48cef98))

### Fixed

- Doctor no longer renames workspace keys on a plain check; only `--fix` (or setup) applies migration. ([4b68fec](https://github.com/Ariestar/sivtr/commit/4b68fec))

## 中文

### 新增

- 新增 `sivtr setup` 一键安装：选择 shell、安装 hooks、迁移 workspace key、把 MCP 装进所选 agent host，再做 smoke check。([2eda454](https://github.com/Ariestar/sivtr/commit/2eda454), [4b68fec](https://github.com/Ariestar/sivtr/commit/4b68fec))
- 新增共享交互式提示（`dialoguer`），供 `setup`、`doctor --fix`、`share` 使用；shell 与 MCP host 支持多选。([4b68fec](https://github.com/Ariestar/sivtr/commit/4b68fec))
- 新增 `sivtr doctor --fix` 与 `sivtr doctor --json`。`--fix` 可创建配置、安装 shell hooks、迁移 workspace key、为已检测 host 安装 MCP；交互模式会先确认再执行。([59aa756](https://github.com/Ariestar/sivtr/commit/59aa756), [4b68fec](https://github.com/Ariestar/sivtr/commit/4b68fec))
- MCP 安装目标扩展到 OpenCode / Pi / Hermes / Cursor，并统一到 `AgentProvider`（`sivtr mcp install --target ...`）。([606907a](https://github.com/Ariestar/sivtr/commit/606907a))
- 新增 workspace key dry-run 检查（`inspect_workspace_keys`），doctor 普通检查只报告、不迁移；`--fix` 才落地。([4b68fec](https://github.com/Ariestar/sivtr/commit/4b68fec))
- 新增贡献指南、微信赞助页，并补充安装/排障文档。([0a5362e](https://github.com/Ariestar/sivtr/commit/0a5362e))

### 变更

- `sivtr migrate` 并入 `sivtr doctor --fix` / `sivtr setup`，独立 migrate 命令已移除。([59aa756](https://github.com/Ariestar/sivtr/commit/59aa756))
- `sivtr init` 不再询问是否启动 Windows Codex hotkey；需要时自行 `sivtr hotkey start`。([4b68fec](https://github.com/Ariestar/sivtr/commit/4b68fec))
- cargo-binstall / 安装脚本默认更偏向 musl Linux 资源。([0a5362e](https://github.com/Ariestar/sivtr/commit/0a5362e))
- 发布矩阵与文档移除 Homebrew；请使用 `cargo binstall`、`install.sh` 或 `install.ps1`。([9bf9651](https://github.com/Ariestar/sivtr/commit/9bf9651))
- 删除残留的 legacy `sift` 兼容路径与文档。([48cef98](https://github.com/Ariestar/sivtr/commit/48cef98))

### 修复

- doctor 普通检查不再重命名 workspace key；只有 `--fix`（或 setup）才会迁移。([4b68fec](https://github.com/Ariestar/sivtr/commit/4b68fec))
