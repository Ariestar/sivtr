# 0.2.5 - 2026-07-12

## English

### Added

- Added cross-device remote memory on a local device daemon: explicit shares (`sivtr share`), mounts (`sivtr remote`), peers (`sivtr peer`), and local workspace origin labels (`sivtr wb`). ([8d0455f](https://github.com/Ariestar/sivtr/commit/8d0455f701d21bd43cc9d7ee791be737da149229), [c8c3cf7](https://github.com/Ariestar/sivtr/commit/c8c3cf7edaca6d61efda33baa8d89996e98f66bc), [1f63a9f](https://github.com/Ariestar/sivtr/commit/1f63a9fd13727c9221030cb05a198235ed2f2bfb))
- Added `origin:body` WorkRefs such as `desk:terminal/...` and `docs:codex/4` so remote and other-local-workspace sources share one selector model. ([2c2d604](https://github.com/Ariestar/sivtr/commit/2c2d604929d2821bd800db4856eb9e452e9eef9a), [4eb35cd](https://github.com/Ariestar/sivtr/commit/4eb35cddee08cbef9ff7dd2669d12adf97d07b4f), [7e1b78b](https://github.com/Ariestar/sivtr/commit/7e1b78b5f0a2bef826889bcf49cf848b96edfc42))
- Added zero-config encrypted peer transport over iroh for remote memory. ([1ee03aa](https://github.com/Ariestar/sivtr/commit/1ee03aa8dac1569531e0d2c03a2a1dde3a6568e9))
- Added bare interactive `sivtr share` as the share entrypoint; it prints a bare invite key on stdout for easy copy/paste. ([d59c025](https://github.com/Ariestar/sivtr/commit/d59c0255d3a2182cd68ba42f86678daae6eafc3e), [fce8e3c](https://github.com/Ariestar/sivtr/commit/fce8e3ce01d011d8e605bfd3ea7bde0d6743eaf9), [ace9574](https://github.com/Ariestar/sivtr/commit/ace957451b0d0dd6a970e88bdf2f993cd2e7d237))
- Added read-only MCP server (`sivtr mcp serve`) with tools `sivtr_search`, `sivtr_show`, `sivtr_zoom`, `sivtr_filter`, and `sivtr_status`, plus host install/uninstall/print-config for Claude Code, Cursor, and Codex. ([710a03e](https://github.com/Ariestar/sivtr/commit/710a03ed1f80a05c1c0f307095cee00e28b71436), [21b37ea](https://github.com/Ariestar/sivtr/commit/21b37eaec9cf8282b41fa92139bbc60948685959))
- Added `sivtr migrate` and run it from `sivtr init` so legacy workspace keys re-key idempotently. ([1bfd2ba](https://github.com/Ariestar/sivtr/commit/1bfd2ba57e6642cfdab9090e6744e1c9e041ee47))
- Added cargo-binstall and install scripts (`install.sh` / `install.ps1`) for release distribution. ([344d58c](https://github.com/Ariestar/sivtr/commit/344d58c9ed4ab5b4a91775962d4bfe92e5ad5696))

### Changed

- Remote memory now uses the device-scoped Share / Grant / Mount model instead of earlier HTTP/pair prototypes. ([8d0455f](https://github.com/Ariestar/sivtr/commit/8d0455f701d21bd43cc9d7ee791be737da149229), [6fa4e4f](https://github.com/Ariestar/sivtr/commit/6fa4e4f4e0d62ef7166f2bce6aaeafa94cce8dd1), [2fc6709](https://github.com/Ariestar/sivtr/commit/2fc67092e2931e5c07ac3cd6136302197dcfb145))
- CLI command modules were regrouped by domain (capture / memory / remote / system) and remote entrypoints were thinned. ([6852dca](https://github.com/Ariestar/sivtr/commit/6852dca8d081eace8ca1e5e783d73e57a5bd4970), [3e986bb](https://github.com/Ariestar/sivtr/commit/3e986bb6b8601b3da5e155617d3e5813400a71e0))
- Workspace record orchestration moved into `sivtr-core` so CLI and remote paths share one query surface. ([5dc6657](https://github.com/Ariestar/sivtr/commit/5dc66571198636f057a4de71d1742d3c120d54e2))
- `sivtr search` always bounds results: when neither `--latest` nor `--limit` is set it now defaults to `--latest 5` (was an implicit `--limit 20`). MCP search inherits the same bound. ([2c8f8ca](https://github.com/Ariestar/sivtr/commit/2c8f8ca87fbdf016a60871afa20fd0190351147a))

### Fixed

- Fixed workspace root resolution by using `std::path::absolute` instead of canonicalize, so missing or non-canonical paths still resolve. ([f948031](https://github.com/Ariestar/sivtr/commit/f9480318fb9fae1acc618f03f91aef264b8e5084))
- Hardened remote share/source paths and dropped the invite key prefix so peer mount commands accept the bare key printed by `sivtr share`. ([2fc6709](https://github.com/Ariestar/sivtr/commit/2fc67092e2931e5c07ac3cd6136302197dcfb145), [ace9574](https://github.com/Ariestar/sivtr/commit/ace957451b0d0dd6a970e88bdf2f993cd2e7d237))
- Improved release workflow retry handling so changelog notes can be reused on manual release retries. ([ce378f5](https://github.com/Ariestar/sivtr/commit/ce378f5c650fc19b2440806f3f9d63e577627b62))

### Documentation

- Documented remote share/mount, `wb` origins, MCP serve/install, and the updated search default across README, docs-site, and the `sivtr-memory` skill. ([0e2805f](https://github.com/Ariestar/sivtr/commit/0e2805f1009d8ced314b5f24314020daaf5488e8), [21b37ea](https://github.com/Ariestar/sivtr/commit/21b37eaec9cf8282b41fa92139bbc60948685959), [2c8f8ca](https://github.com/Ariestar/sivtr/commit/2c8f8ca87fbdf016a60871afa20fd0190351147a))

## 中文

### 新增

- 新增跨设备远程记忆：本机 device daemon、显式 share（`sivtr share`）、mount（`sivtr remote`）、peer（`sivtr peer`）以及本机 workspace origin 标签（`sivtr wb`）。([8d0455f](https://github.com/Ariestar/sivtr/commit/8d0455f701d21bd43cc9d7ee791be737da149229), [c8c3cf7](https://github.com/Ariestar/sivtr/commit/c8c3cf7edaca6d61efda33baa8d89996e98f66bc), [1f63a9f](https://github.com/Ariestar/sivtr/commit/1f63a9fd13727c9221030cb05a198235ed2f2bfb))
- 新增 `origin:body` WorkRef（如 `desk:terminal/...`、`docs:codex/4`），远程与其他本机 workspace 共用同一套 selector 模型。([2c2d604](https://github.com/Ariestar/sivtr/commit/2c2d604929d2821bd800db4856eb9e452e9eef9a), [4eb35cd](https://github.com/Ariestar/sivtr/commit/4eb35cddee08cbef9ff7dd2669d12adf97d07b4f), [7e1b78b](https://github.com/Ariestar/sivtr/commit/7e1b78b5f0a2bef826889bcf49cf848b96edfc42))
- 新增基于 iroh 的零配置加密 peer 传输。([1ee03aa](https://github.com/Ariestar/sivtr/commit/1ee03aa8dac1569531e0d2c03a2a1dde3a6568e9))
- 裸 `sivtr share` 成为交互式分享入口，并把 bare invite key 打印到 stdout 方便复制。([d59c025](https://github.com/Ariestar/sivtr/commit/d59c0255d3a2182cd68ba42f86678daae6eafc3e), [fce8e3c](https://github.com/Ariestar/sivtr/commit/fce8e3ce01d011d8e605bfd3ea7bde0d6743eaf9), [ace9574](https://github.com/Ariestar/sivtr/commit/ace957451b0d0dd6a970e88bdf2f993cd2e7d237))
- 新增只读 MCP server（`sivtr mcp serve`），工具：`sivtr_search`、`sivtr_show`、`sivtr_zoom`、`sivtr_filter`、`sivtr_status`；并支持 Claude Code / Cursor / Codex 的 install、uninstall、print-config。([710a03e](https://github.com/Ariestar/sivtr/commit/710a03ed1f80a05c1c0f307095cee00e28b71436), [21b37ea](https://github.com/Ariestar/sivtr/commit/21b37eaec9cf8282b41fa92139bbc60948685959))
- 新增 `sivtr migrate`，并在 `sivtr init` 中自动运行，幂等迁移旧 workspace key。([1bfd2ba](https://github.com/Ariestar/sivtr/commit/1bfd2ba57e6642cfdab9090e6744e1c9e041ee47))
- 新增 cargo-binstall 与安装脚本（`install.sh` / `install.ps1`）用于发布分发。([344d58c](https://github.com/Ariestar/sivtr/commit/344d58c9ed4ab5b4a91775962d4bfe92e5ad5696))

### 变更

- 远程记忆改为 device 级 Share / Grant / Mount 模型，替换早期 HTTP/pair 原型。([8d0455f](https://github.com/Ariestar/sivtr/commit/8d0455f701d21bd43cc9d7ee791be737da149229), [2fc6709](https://github.com/Ariestar/sivtr/commit/2fc67092e2931e5c07ac3cd6136302197dcfb145))
- CLI 命令按 capture / memory / remote / system 领域重组，并精简 remote 入口。([6852dca](https://github.com/Ariestar/sivtr/commit/6852dca8d081eace8ca1e5e783d73e57a5bd4970), [3e986bb](https://github.com/Ariestar/sivtr/commit/3e986bb6b8601b3da5e155617d3e5813400a71e0))
- Workspace record 编排下沉到 `sivtr-core`，CLI 与 remote 共用同一查询面。([5dc6657](https://github.com/Ariestar/sivtr/commit/5dc66571198636f057a4de71d1742d3c120d54e2))
- `sivtr search` 始终有界：未设 `--latest`/`--limit` 时默认 `--latest 5`（此前隐式 `--limit 20`）；MCP search 继承同一默认。([2c8f8ca](https://github.com/Ariestar/sivtr/commit/2c8f8ca87fbdf016a60871afa20fd0190351147a))

### 修复

- 加固 remote share/source 路径，并去掉 invite key 前缀，使 peer 可直接粘贴 `sivtr share` 打印的 bare key。([2fc6709](https://github.com/Ariestar/sivtr/commit/2fc67092e2931e5c07ac3cd6136302197dcfb145), [ace9574](https://github.com/Ariestar/sivtr/commit/ace957451b0d0dd6a970e88bdf2f993cd2e7d237))
