# 0.2.4 - 2026-06-10

## English

### Added

- Added `sivtr filter`, a dedicated WorkSet filtering command that works with source selectors, saved WorkSets, and WorkSets piped through stdin (`@`). ([9fe7018](https://github.com/Ariestar/sivtr/commit/9fe7018edf1cd7b2906eb751e81f48795260f433))
- Added `sivtr var` for saved WorkSet variables, including commands to create, merge, drop entries from, remove, and clean up saved WorkSets. ([dae5a01](https://github.com/Ariestar/sivtr/commit/dae5a01a7d2db942d52f55d30a17d248be516219))
- Added `sivtr var list` so saved WorkSet variables can be inspected before reuse. ([a7f25be](https://github.com/Ariestar/sivtr/commit/a7f25be023e1a33891fcc9ab24315d473c1564b3))
- Added `sivtr nav` for deterministic anchor movement across parent, child, sibling, range, and current-anchor selections. ([0af31bf](https://github.com/Ariestar/sivtr/commit/0af31bf2fbbb7d1ee95bab2fcfd084b8ecb8f7d1))
- Added Hermes as an agent-session provider for copy, picker, search, refs, and WorkSet workflows. ([ffd869e](https://github.com/Ariestar/sivtr/commit/ffd869e767a5cb8861337b4f3f138cf3fc8a5f3d))
- Added global `--color auto|always|never` handling for status and diagnostic output. ([00d29fd](https://github.com/Ariestar/sivtr/commit/00d29fdb0f7f0c852c052cabe0cf6efae8287ee2))

### Changed

- Standardized CLI status output so user-facing diagnostics follow a consistent format. ([00d29fd](https://github.com/Ariestar/sivtr/commit/00d29fdb0f7f0c852c052cabe0cf6efae8287ee2))
- Kept status and diagnostic messages out of stdout so command output remains safe to pipe into other tools. ([00d29fd](https://github.com/Ariestar/sivtr/commit/00d29fdb0f7f0c852c052cabe0cf6efae8287ee2))
- Moved `search` and `work parts` onto the shared WorkSet filtering implementation. ([9fe7018](https://github.com/Ariestar/sivtr/commit/9fe7018edf1cd7b2906eb751e81f48795260f433))
- Reused the agent-copy help text across CLI surfaces to keep command guidance consistent. ([a8c357f](https://github.com/Ariestar/sivtr/commit/a8c357f45638b2342b4fa75a45322c8de53862df))

### Fixed

- Fixed workspace log resolution during `flush` so captured output is attached to the intended workspace. ([eda6708](https://github.com/Ariestar/sivtr/commit/eda67089142f2d4f92ed741e5bc6a51727f590a8))
- Fixed Windows Nushell integration by preserving the rendered prompt before `flush`, avoiding empty captures when Nushell does not expose prompt updates to later hooks. ([be2b1a3](https://github.com/Ariestar/sivtr/commit/be2b1a34e1644be51e5a40cc181f1bf96beea14e))
- Fixed Claude workspace-session matching across local clones that share the same Git remote. (thanks [@Jacobinwwey](https://github.com/Jacobinwwey), [22b1e11](https://github.com/Ariestar/sivtr/commit/22b1e110f937277f1fe1ebb644e104582bcc16ed))
- Hardened AI workspace repository matching so sessions from equivalent local checkouts are found more reliably. (thanks [@Jacobinwwey](https://github.com/Jacobinwwey), [2bd2db2](https://github.com/Ariestar/sivtr/commit/2bd2db21b0db6d2c0c445729822bc2bdb298deb3))
- Skipped invalid unrelated Claude session files during session listing so one bad file no longer breaks copy, picker, or work-record flows. (thanks [@Jacobinwwey](https://github.com/Jacobinwwey), [96af006](https://github.com/Ariestar/sivtr/commit/96af006cf2386cdbe35900b6be13050d039a6b02))

## 中文

### 新增

- 新增 `sivtr filter`，作为专门的 WorkSet 过滤命令，可用于 source selector、已保存的 WorkSet，以及通过 stdin 传入的 `@` WorkSet。([9fe7018](https://github.com/Ariestar/sivtr/commit/9fe7018edf1cd7b2906eb751e81f48795260f433))
- 新增 `sivtr var`，用于管理已保存的 WorkSet 变量，支持创建、合并、移除部分条目、删除和清理。([dae5a01](https://github.com/Ariestar/sivtr/commit/dae5a01a7d2db942d52f55d30a17d248be516219))
- 新增 `sivtr var list`，可以在复用前查看已保存的 WorkSet 变量。([a7f25be](https://github.com/Ariestar/sivtr/commit/a7f25be023e1a33891fcc9ab24315d473c1564b3))
- 新增 `sivtr nav`，用于在父级、子级、相邻节点、范围和当前 anchor 之间进行确定性移动。([0af31bf](https://github.com/Ariestar/sivtr/commit/0af31bf2fbbb7d1ee95bab2fcfd084b8ecb8f7d1))
- 新增 Hermes agent-session provider，支持 copy、picker、search、refs 和 WorkSet 工作流。([ffd869e](https://github.com/Ariestar/sivtr/commit/ffd869e767a5cb8861337b4f3f138cf3fc8a5f3d))
- 新增全局 `--color auto|always|never`，用于控制 status 和 diagnostic 输出的颜色。([00d29fd](https://github.com/Ariestar/sivtr/commit/00d29fdb0f7f0c852c052cabe0cf6efae8287ee2))

### 变更

- 统一 CLI status 输出格式，让面向用户的诊断信息更一致。([00d29fd](https://github.com/Ariestar/sivtr/commit/00d29fdb0f7f0c852c052cabe0cf6efae8287ee2))
- 将 status 和 diagnostic 信息排除在 stdout 之外，让命令输出可以稳定地接入其他工具。([00d29fd](https://github.com/Ariestar/sivtr/commit/00d29fdb0f7f0c852c052cabe0cf6efae8287ee2))
- 将 `search` 和 `work parts` 迁移到共享的 WorkSet 过滤实现。([9fe7018](https://github.com/Ariestar/sivtr/commit/9fe7018edf1cd7b2906eb751e81f48795260f433))
